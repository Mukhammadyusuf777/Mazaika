# 🏆 Документация платформы Mazaika (No-Code конструктор ботов и Mini Apps)

Добро пожаловать в полную техническую и пользовательскую документацию платформы **Mazaika** — современного SaaS-решения для визуального проектирования Telegram-ботов и автоматического развертывания встроенных сайтов (Mini Apps / Web Apps) для бизнеса в Узбекистане.

---

## 1. 🏗 Общее описание и миссия проекта

**Mazaika** — это low-code/no-code экосистема, которая позволяет предпринимателям и маркетологам собирать сложные сценарии обслуживания клиентов, интернет-магазины и системы автоматизации в Telegram без привлечения разработчиков и написания кода.

### Ключевые возможности:
* **Визуальный drag-and-drop редактор графов** сценариев на базе React Flow.
* **Единый конструктор сайтов и Telegram Mini Apps**: редактируете в одном месте (No-Code Constructor) — изменения мгновенно публикуются и на Web-сайте, и в Mini App внутри Telegram-бота.
* **Локальные интеграции**: встроенные модули оплаты Payme и Click, а также интеграция с CRM-системами GetCourse и Yclients.
* **Встроенные CRM-чаты**: общение с подписчиками в реальном времени прямо из панели управления платформы.
* **Умные переменные и вычисления**: поддержка математических формул и кастомного JavaScript прямо внутри сценария бота.
* **Асинхронные задержки (Timer)**: корпоративная система планирования рассылок и таймеров, устойчивая к перезагрузке серверов.

## 2. 💻 Архитектура и стек технологий

Платформа спроектирована по принципу монорепозитория, разделенного на клиентскую и серверную части:

### Клиентская часть (`mazaika-app`):
* **Библиотека**: React 18, Vite.
* **Язык**: TypeScript (строгая типизация).
* **Стейт-менеджер**: Zustand (для хранения авторизации и состояний визуального редактора).
* **Графовый движок**: `@xyflow/react` (React Flow) для рендеринга и управления узлами сценариев.
* **Стилизация**: Ванильный CSS. Использован кастомный дизайн-код **"Aurora Dark"** (глубокие темные фоны с неоновыми свечениями, Glassmorphism, микроанимации и полная отзывчивость на мобильных телефонах).

### Серверная часть (`mazaika-backend`):
* **Фреймворк**: NestJS (модульная архитектура, TypeScript).
* **Движок Telegram**: `Telegraf` для динамического запуска и остановки ботов "на лету" по токенам клиентов.
* **База данных**: PostgreSQL (в продакшене) / SQLite (для легковесного локального тестирования).
* **ORM**: Prisma ORM для безопасной работы с базой данных и автоматических миграций.

## 4. 🗄 Схема базы данных (Prisma Schema)

Взаимодействие с базой данных осуществляется через Prisma. Ниже приведено описание ключевых сущностей:

### 1. `User` (Пользователи)
Хранит учетные данные владельцев аккаунтов на платформе (Email, Phone, Name, Password). Связан с ботами (`Bot[]`).

### 2. `Bot` (Telegram-боты)
Содержит информацию о подключенных Telegram-ботах:
* `name` — Имя бота.
* `token` — API Токен (уникальный ключ из BotFather).
* `status` — Текущий статус работы (`active` / `stopped`).
* `userId` — Связь с создателем (`User`).
* Связан со сценариями (`Workflow[]`), подписчиками (`Contact[]`) и вебхуками (`Webhook[]`).

### 3. `Workflow` (Сценарии диалогов)
Хранит графы логики:
* `nodes` (String/Json) — Массив узлов (позиции, тексты, типы блоков).
* `edges` (String/Json) — Массив связей (какая кнопка ведет на какой блок).
* `isMain` — Флаг основного сценария бота.

### 4. `Contact` (Подписчики бота / CRM)
Хранит данные людей, запустивших бота в Telegram:
* `telegramId` — Уникальный ID пользователя в Telegram.
* `firstName`, `lastName`, `username` — Профиль из Telegram.
* `state` — Текущее состояние диалога (какой блок сейчас проходит клиент) и словарь сохраненных переменных (`{ name: "Иван", age: "25" }`).

### 5. `Message` (История чатов)
Хранит лог переписки для CRM-раздела:
* `text` — Текст сообщения.
* `direction` — Направление (`inbound` — от пользователя, `outbound` — от оператора/бота).
* `contactId` — Связь с профилем клиента (`Contact`).

### 6. `Webhook` (Внешние интеграции)
Позволяет пользователю настроить отправку данных на сторонние сервисы при срабатывании узла сценария.

## 5. 🛠 Описание модулей и функционала

### 5.1. Конструктор сценариев (Visual Flow Editor)
Это ядро платформы. Пользователь создает блок-схему, соединяя узлы линиями.

#### Доступные типы блоков (22 вида):
1. **Boshlash (Start)**: Стартовая точка входа. Реагирует на `/start`. Считывает реферальные метки (`t.me/bot?start=promo_12`) и пишет их в переменную `{start_payload}`.
2. **Xabar (Message)**: Отправка текста, картинок или видео. Поддерживает подстановку переменных (например, `Привет, {name}!`) и создание inline-кнопок.
3. **Zanjir (Chain)**: Моментальный автоматический переход на другой блок без ожидания ввода от пользователя (с защитой от вечных циклов).
4. **Kechikish (Timer)**: Задержка выполнения сценария. Короткие задержки обрабатываются в памяти, длинные (более 5 сек) записываются в базу данных — бот не забудет отправить сообщение, даже если сервер перезагрузится.
5. **Matnli savol (Question)**: Задает вопрос и сохраняет текстовый ответ пользователя в указанную переменную.
6. **Telefon raqam (Phone)**: Запрашивает телефон через системную кнопку Telegram или парсит ручной ввод пользователя (с валидацией формата).
7. **Email manzil (Email)**: Запрашивает почту, валидирует по регулярному выражению и приводит к нижнему регистру.
8. **Lokatsiya (Location)**: Запрашивает геопозицию и записывает GPS-координаты в переменную (удобно для доставки).
9. **Shart (If/Condition)**: Разветвляет логику. Сравнивает переменные (равно, не равно, больше, меньше, содержит, регулярное выражение).
10. **Kanalga a'zolik (Subscription)**: Проверяет, подписан ли пользователь на указанный канал/группу (бот должен быть админом в этом канале).
11. **O'zgaruvchi (Variable Set)**: Записывает значение в переменную. Поддерживает математические операции (например: `{points} + 5` или `{price} * 0.9`).
12. **O'zg. O'chirish (Delete Variable)**: Удаляет одну или список переменных (через запятую) из памяти контакта.
13. **A/B Test**: Распределяет трафик пользователей по двум веткам в заданном соотношении (например, 80/20) с записью группы в переменную для аналитики.
14. **JS Hisoblagich (JS Calculator)**: Выполняет любой валидный код на JavaScript с автоматическим доступом к переменным контакта.
15. **HTTP So'rov / Webhook**: Отправляет GET/POST запросы к внешним API. Поддерживает парсинг сложных JSON-ответов через JSON-путь (например, `data.weather.temp`) в переменную.
16. **Payme / Click**: Выставляет официальный инвойс Telegram на оплату. Бот блокирует переход дальше по сценарию, пока оплата не будет успешно проведена.

### 5.2. Конструктор сайтов & Telegram Mini Apps (Site Builder)
Уникальный визуальный конструктор, синхронизированный с Telegram WebApp:
* **Единый источник правды**: Настройки темы (`theme`), цветовой гаммы (`themeColor`) и структуры блоков сохраняются в единый документ `/bots/{botId}/site/config` в Firestore.
* **Блоки No-Code**: Пользователь может добавлять, удалять и редактировать следующие блоки:
  * **Hero (Баннер)**: Слайдер/заголовок, подзаголовок, фоновое изображение и кнопка действия (CTA).
  * **About (О нас)**: Информационный блок с текстом о бизнесе.
  * **Catalog (Магазин/Товары)**: Карточки товаров с изображениями, описаниями и ценами.
  * **Contacts (Контакты)**: Форма обратной связи, адрес и социальные сети.
* **Дизайн-системы**: Доступны готовые стили: минималистичный светлый (*Minimalist*), темный неоновый (*Neon Cyberpunk*) и стеклянный (*Glassmorphism*).
* **Симулятор**: В личном кабинете справа отображается интерактивный iframe-симулятор телефона, который в режиме реального времени показывает, как сайт выглядит на мобильном.
* **Telegram-Интеграция**: Кнопка «Menu» в самом боте автоматически настраивается на открытие этого Mini App (`https://mazaika.pages.dev/site/${botId}`).

---

### 5.3. CRM-раздел и чаты
* **Контакты (Contacts)**: Список всех пользователей бота, поиск по имени/Telegram ID, просмотр переменных контакта и кнопка экспорта базы.
* **CRM Чаты (Real-time Chats)**: Оператор видит переписку бота с клиентом и может в любой момент вмешаться, написав сообщение прямо из браузера.
* **Аналитика**: Графики роста базы подписчиков и активности бота за последние дни.

## 6. 🚀 Инструкция по установке и локальному запуску

### Шаг 1. Клонирование и настройка окружения
Убедитесь, что у вас установлены Node.js (v18+) и Docker.

Создайте `.env` файл в директории `mazaika-backend/`:
```env
PORT=3001
DATABASE_URL="postgresql://mazaika_user:mazaika_password@localhost:5432/mazaika?schema=public"
DIRECT_URL="postgresql://mazaika_user:mazaika_password@localhost:5432/mazaika?schema=public"
JWT_SECRET="your-super-secret-key"
```

### Шаг 2. Запуск инфраструктуры (БД и Redis)
Запустите контейнеры PostgreSQL и Redis в корне проекта:
```bash
docker-compose up -d
```

### Шаг 3. Установка зависимостей и запуск Бэкенда
```bash
cd mazaika-backend
npm install
npx prisma db push     # Синхронизация схемы БД с PostgreSQL
npm run start:dev      # Запуск NestJS в режиме разработки
```

### Шаг 4. Установка зависимостей и запуск Фронтенда
```bash
cd ../mazaika-app
npm install
npm run dev            # Запуск Vite сервера разработки
```

Приложение будет доступно по адресу `http://localhost:5173`.

## 7. 📈 Журнал улучшений и адаптивности (Changelog)

В ходе последних этапов разработки были проведены масштабные работы по полировке дизайна и исправлению багов:

1. **Мобильная адаптивность (Mobile UX)**:
   * **Боковое меню**: На смартфонах боковая панель сворачивается в скользящий Drawer-панель, вызываемую по кнопке-гамбургеру.
   * **CRM Чаты**: На мобильных экранах список чатов и активный диалог не сжимаются вместе. Экран разделен: клик на контакт открывает чат на весь экран, а кнопка «← Назад» возвращает оператора к списку чатов (устранен баг сжатия панели чата).
   * **Конструктор (Site Builder)**: Шапка конструктора и панель режимов `Desktop/Mobile` переведены в адаптивную CSS-сетку. На телефонах они выстраиваются в вертикальную стопку, не наезжая друг на друга.
   * **Таблицы**: Добавлена горизонтальная прокрутка для списков вебхуков и контактов на мобильных устройствах, предотвращающая выход таблиц за границы экрана.
2. **Исправление визуального редактора графов (React Flow)**:
   * **Стыковка связей**: Из кастомных портов (Handles) узлов были удалены неиспользуемые ID (`in`/`out`). Это позволило линиям связей из шаблонов по умолчанию притягиваться ровно к круглым точкам-коннекторам на границах блоков, вместо того чтобы пробивать блоки насквозь и подключаться к их центру.
   * **Закрытие сайдбара**: В редакторе сценариев добавлена кнопка `✕`, позволяющая скрыть боковую панель блоков на смартфонах и ПК, освобождая максимум места для работы с холстом.
3. **Синхронизация Landing Page**:
   * Интерактивный демонстрационный граф на главной странице сайта переведен на процентные координаты (`left` и `width` для блоков). Теперь при открытии сайта на любых мониторах и разрешениях SVG-линии соединений и круглые порты на границах блоков остаются жестко соединенными друг с другом.
